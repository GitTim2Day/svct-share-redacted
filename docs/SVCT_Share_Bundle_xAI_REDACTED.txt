7. SVCT vs BLOCKCHAIN (efficiency orientation)
================================================================================

  Blockchain typical:  global consensus, miner/validator cost, block time latency,
                         immutable ledger optimized for trust between strangers.

  SVCT spine:            single-operator local append-only hash chain;
                         verify-on-read before checkpoint; no mining;
                         64-cell rows optimized for recall + provenance witness,
                         not currency or smart-contract execution.

  Efficiency hooks:
  · content_hash dedup before append
  · large_bypass for X>50000 codepoint payloads
  · TextCodec + WordLibraryCodec (LIB_MARK lossless) for text
  · numeric_hints → Tim's Table snap → cell fill
  · sameness hierarchy (header/stub/true_dupe/unique) on prefilter
  · free-first cost routing before paid API (see §8)
